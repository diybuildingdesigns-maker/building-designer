# DIY Building Designs Professional
# MBS_PAIN_POINTS.md

## Purpose

This document captures every frustration, inefficiency, and limitation experienced while using MBS.

Every pain point should lead to a better feature in DIY Building Designs Professional.

---

# Core Rule

If MBS requires multiple steps to perform a common task, DIY Building Designs Professional should accomplish it with the fewest possible steps, preferably directly from the 3D model.

---

# Current Pain Points

## 1. Resizing a Building

### MBS

Changing the building width, length, or height requires editing multiple settings and often rebuilding parts of the project manually.

### DIY Goal

Resize the building directly from the 3D model or Properties panel.

Expected time:

**Less than 5 seconds**

Automatic updates:

- Frame locations
- Bay spacing
- Girts
- Purlins
- Bracing
- Panels
- Material quantities
- Pricing
- Shop drawings

---

## 2. Complicated User Interface

### MBS

Functions are spread across many windows, dialogs, and hidden settings.

### DIY Goal

One modern workspace with:

- 3D model
- Properties panel
- Project tree
- Toolbar

Everything should be discoverable without searching through menus.

---

## 3. Manual Regeneration

### MBS

Changes often require rebuilding or regenerating information.

### DIY Goal

Changes regenerate automatically in the background.

No "Rebuild" button for normal edits.

---

## 4. Steep Learning Curve

### MBS

New users struggle to learn where commands are located.

### DIY Goal

A first-time user should be able to create a basic building with minimal training.

---

## 5. Information Stored in Too Many Places

### MBS

Building data is scattered across different dialogs and files.

### DIY Goal

One Building object stores all project information.

Everything else is generated from it.

---

# Future Pain Points

As development continues, every new frustration should be documented here.

# Guiding Question

Whenever we design a feature, ask:

"Is this easier than MBS?"

If the answer is "no," redesign it.
