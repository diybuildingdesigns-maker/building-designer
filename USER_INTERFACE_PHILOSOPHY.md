# DIY Building Designs Professional

# User Interface Philosophy v1.0

## Purpose

The purpose of this document is to establish the guiding principles for
every screen, menu, and workflow in DIY Building Designs Professional.

The software must always favor simplicity, speed, and visual interaction
over complex menus or hidden settings.

## Core Philosophy

The 3D design interface is the primary control center of the
application.

Users should never need to search through folders, edit hidden files, or
navigate complicated dialog boxes to make common building changes.

Every major building modification should be possible directly from the
3D model or from a simple properties panel.

## Design Principles

-   One model controls everything.
-   The building is edited visually.
-   Changes update instantly whenever possible.
-   Keep the number of clicks to a minimum.
-   Show only the options needed for the current task.
-   Advanced settings belong in an Advanced section, not on the main
    screen.
-   The interface should be understandable by someone who has never used
    steel building software.

## Examples

Changing any of the following should take only a few seconds:

-   Building width
-   Building length
-   Eave height
-   Roof pitch
-   Bay spacing
-   Door locations
-   Window locations
-   Frame spacing
-   Roof and wall colors

After a change, the software should automatically update:

-   3D model
-   Structural framing
-   Material quantities
-   Weight calculations
-   Pricing
-   Shop drawings
-   Reports

## Long-Term Goal

DIY Building Designs Professional should feel like modern CAD software
while remaining approachable for builders, designers, and sales
staff---not just experienced draftsmen.
